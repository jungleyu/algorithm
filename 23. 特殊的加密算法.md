# 特殊的加密算法

## 题目描述

有一种特殊的加密算法，明文为一段数字串，经过密码本查找转换，生成另一段密文数字串。

**加密规则**：
- 明文为一段由 `0~9` 组成的数字串
- 密码本为 `0~9` 组成的二维数组

**匹配规则**：
- 需要按明文串的数字顺序在密码本里找到同样的数字串
- 密码本里的数字串是由**相邻的单元格数字**组成
- **相邻定义**：上下和左右是相邻的，**对角线不相邻**
- **同一个单元格的数字不能重复使用**

**加密规则**：
- 每一位明文对应密文即为密码本中找到的单元格所在的**行号和列号**（序号从0开始）
- 例如：明文第 i 位 `Data[i]` 对应密码本单元格 `Book[x][y]`，则密文为 `X Y`（X和Y之间用空格隔开）

**特殊情况**：
- 如果有多条密文，返回**字符序最小**的密文
- 如果密码本无法匹配，返回 `error`

## 输入描述

第一行输入 1 个正整数 `N`，代表明文的长度（`1 ≤ N ≤ 200`）

第二行输入 N 个明文组成的序列 `Data[i]`（`0 ≤ Data[i] ≤ 9`）

第三行输入 1 个正整数 `M`，代表密码本矩阵的大小

接下来 M 行，每行 M 个数，代表密文矩阵

## 输出描述

输出字典序最小的密文，如果无法匹配，输出 `error`

## 示例

### 示例1

**密码本**
```
0 0 2
1 3 4
6 6 4
```

**明文**
```
3
```

**密文**
```
1 1
```

**说明**
数字3在密码本第1行第1列

---

### 示例2

**密码本**
```
0 0 2
1 3 4
6 6 4
```

**明文**
```
0 3
```

**密文**
```
0 1 1 1
```

**说明**
- 第一个0：第0行第1列 → `0 1`
- 第二个3：第1行第1列 → `1 1`
- 合并：`0 1 1 1`

---

### 示例3

**密码本**
```
0 0 2 4
1 3 4 6
3 4 1 5
6 6 6 5
```

**明文1**
```
0 0 2 4
```

**密文**
```
0 0 0 1 0 2 0 3  和  0 0 0 1 0 2 1 2
```

**输出**
```
0 0 0 1 0 2 0 3
```

**说明**
- 两条路径都匹配，取字典序最小的
- `0 0 0 1 0 2 0 3` < `0 0 0 1 0 2 1 2`

**明文**
