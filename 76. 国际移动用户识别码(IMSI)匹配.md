# 国际移动用户识别码(IMSI)匹配 算法题
## 一、题目描述
### 1. 概念说明
**IMSI（国际移动用户识别码）**：是全球唯一的移动用户标识，由一串数字组成，常用于通信网络中识别用户。
本题基于**前缀匹配**规则实现IMSI号码的匹配逻辑。

### 2. 匹配规则
1. 系统中预先存储多条**IMSI规则条目**，每条条目包含**IMSI前缀**和对应的归属信息；
2. 对待检测的手机号/IMSI号码，从所有规则中寻找**最长可匹配前缀**；
3. 若存在多条相同长度的最长匹配前缀，按规则录入**先后顺序**，取第一条匹配结果；
4. 若无任何前缀匹配，则判定为匹配失败。

### 3. 输入说明
- 第一行：输入整数 $N$，代表预先存储的IMSI规则条数；
- 接下来 $N$ 行：每行输入**IMSI前缀字符串** + 空格 + **归属描述字符串**，为一条规则；
- 再下一行：输入整数 $M$，代表待查询的IMSI号码个数；
- 接下来 $M$ 行：每行输入一个待查询的IMSI数字字符串。

### 4. 输出说明
针对每一个待查询IMSI号码：
- 找到**最长匹配前缀**对应的归属描述并输出；
- 无匹配前缀时，统一输出 `No Match`。

## 二、输入输出示例（补充常规样例）
### 样例输入
```
3
46000 中国移动
46001 中国联通
460 国内运营商
2
46000123456789
13800138000
```

### 样例输出
```
中国移动
No Match
```

### 样例解析
1. 第一个查询号码 `46000123456789`：
   可匹配前缀：`460`、`46000`；最长前缀为 `46000`，输出对应归属「中国移动」。
2. 第二个查询号码 `13800138000`：无任何前缀匹配，输出 `No Match`。

## 三、解题思路
1. **数据存储**：使用列表按录入顺序保存所有`(前缀, 归属信息)`规则，保证顺序优先级；
2. **遍历匹配**：对每个查询号码，遍历全部规则，记录**能匹配的前缀长度**与对应归属；
3. **筛选最优解**：
   - 遍历完成后，找出所有匹配结果中**长度最大**的前缀；
   - 若有多个同最长长度前缀，因列表有序，第一个遍历到的即为答案；
4. **结果输出**：存在匹配则输出归属，否则输出`No Match`。

## 四、核心考点
1. 字符串**前缀匹配**逻辑；
2. 遍历筛选最值（最长前缀）；
3. 保留原始顺序实现**同长度优先录入靠前规则**；
4. 基础字符串处理。
