# 200分 产品模块算法检验
时间限制:1秒 空间限制:262MB

## 题目描述
在产品配置中,一个配置产品是由多个产品模块(CM)构成,每个CM有自身的算法,且模块间可能存在算法依赖。例如电脑产品是由主板、CPU、显卡等CM构成。CPU模块(CM1)算法依赖主板模块(CM2)算法,记作CM2<-CM1,算法引擎会通过算法依赖确保此前后CM执行的顺序。如果存在模块算法循环依赖的场景,那么算法引擎会报警。

## 输入描述
输入行为依赖情况,例如:CM3<-CM2

## 输出描述
计算出循环依赖的CM数量

## 示例1
### 输入
```
CM5<-CM3,CM4<-CM5,CM6<-CM4,CM6<-CM1,CM5<-CM6
```
### 输出
```
3
```

## 示例2
### 输入
```
CM5<-CM3,CM3<-CM6,CM6<-CM4,CM3<-CM4,CM4<-CM1
```
### 输出
```
0
```
